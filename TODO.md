# Agor iOS — Bugs & Feature Requests

## Feature Requests

- [ ] **Cache session list for instant display on launch** — Persist the last-fetched boards/worktrees/sessions to disk (e.g. UserDefaults or a local JSON file) so the sidebar shows cached data immediately on app open, then refreshes from the API in the background. Avoids the empty-list flash on every launch.
- [ ] **Show reconnecting/loading indicator when returning from background** — When the app wakes from sleep, the socket reconnects and data re-fetches, but there's no visual feedback. Show a banner or spinner (e.g. "Reconnecting..." → "Updating..." → done) so the user knows the app is catching up. Should appear at the top of the chat or as an overlay, and dismiss automatically once the socket is reconnected and messages are refreshed.
- [ ] **Notification when session finishes (running → idle)** — Show a local notification when a session transitions from `running` to `idle`. Should work for: (1) favorited/pinned sessions, and (2) the last session the user was viewing. This way you know your agent is done even if you switched to another app. Only fire when the app is backgrounded or user is viewing a different session.
- [ ] **File browser for worktrees** — Add a file manager view accessible from a worktree or session. Uses the existing `GET /file` API to browse directories and view files. Features: (1) Directory listing with folders and files, navigate in/out. (2) View text files and markdown files with rendered formatting. (3) View images inline (png, jpg, gif, svg). (4) Syntax-highlighted code files (or at minimum monospaced display). Read-only is fine for v1. Access points: (a) long-press / context menu on a worktree or session row in sidebar → "Browse Files", (b) toolbar button in the open chat view (e.g. folder icon in the chat header bar).
- [ ] **Use icons instead of colored dots for session status** — Replace the current text + colored dot `StatusBadge` with SF Symbol icons. E.g.: idle = checkmark.circle, running = progress.indicator / arrow.trianglehead.2.clockwise (animated spin), awaiting_permission = lock.fill, awaiting_input = questionmark.circle, failed = xmark.circle, completed = checkmark.circle.fill, stopping = stop.circle. Icons are more scannable and take less space than text labels.
- [ ] **Merge status icon with stop button when running** — When a session is `running` and connected, the status icon in the chat header should double as a stop button (e.g. stop.fill icon, tappable). Tapping it calls `POST /sessions/:id/stop` to send SIGTERM. When not running, show the regular status icon (non-tappable). This saves space and puts the stop action right where you're already looking.

- [ ] **Basic account/connection controls** — Missing essential controls: (1) Logout button (clears JWT + refresh token from Keychain, disconnects socket, returns to ConnectionSetupView). (2) Switch server / edit daemon URL. (3) Show current connection info (daemon URL, logged-in user email). Could live in a Settings/Account screen accessible from the sidebar footer or a gear icon.
- [ ] **Clean/reset session button** — Add a "Clean" or "Reset" button next to the existing archive button in the session actions. Should clear the session's message history and reset it to a fresh state (like starting a new conversation in the same session). Useful when a session gets cluttered or stuck.
- [ ] **Smart URL handling on connect** — The connection setup should be forgiving with URL input: (1) If user enters just an IP like `192.168.1.100`, auto-try `http://192.168.1.100:3030`. (2) Try both `http://` and `https://` regardless of what the user typed — try the one they entered first, fall back to the other. (3) Hit `/health` endpoint to validate the connection before attempting login. (4) If the URL has a path like `/ui`, strip it and use just the host:port (the daemon root). This way the user can paste any Agor URL and it just works.

## Bugs

- [ ] **Claude agent icon should be a star** — The AgentIcon for `claude-code` should display a star icon, not whatever it currently uses.
- [ ] **Session list not refreshed on expand/collapse** — When expanding a worktree in the sidebar, the session list should be re-fetched from the API (new sessions may have been created since last load). Currently it only loads once and goes stale. Also add periodic polling on the boards/worktrees/sessions level (e.g. every 30-60s) so the sidebar stays fresh even without user interaction.
- [ ] **Tool use/result blocks sometimes can't be expanded** — Collapsible tool use and tool result blocks sometimes don't respond to tap, making it impossible to see the command or output. Might be a gesture conflict or hit target issue. Need to verify the DisclosureGroup/tap area is large enough and not blocked by another view.
- [ ] **Scheduled sessions should not appear in "Needs Attention"** — Sessions created by the scheduler have titles starting with `[Scheduled `. These should be excluded from the "Needs Attention" section in the sidebar. Filter them out by checking `session.scheduled_from_worktree == true` or by title prefix `[Scheduled `.
- [ ] **"Needs Attention" list should show board icon instead of agent icon** — In the important/attention session list, replace the agent model icon with the board icon/emoji. Since sessions there are from different boards, showing which board they belong to is more useful for context than which agent type they use.

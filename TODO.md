# Agor iOS — Bugs & Feature Requests

## Bugs

- [x] **HTTPS URLs get port 3030 appended, breaking reverse proxy setups** — In `AuthService.swift` line 61, `let port = components.port ?? 3030` always defaults to 3030 even for HTTPS URLs. When user enters `https://agor-rmi.duckdns.org/`, it becomes `https://agor-rmi.duckdns.org:3030` which fails because the daemon is behind a reverse proxy on port 443. Fix: only default to `:3030` when scheme is `http` (local dev). For `https`, omit the port entirely (443 is implied). The web UI works because `getDaemonUrl()` in `daemon.ts` uses `window.location.origin` which doesn't append a port for standard HTTPS.
- [x] **File browser fails with "Session expired. Please log in again."** — The iOS app used REST (URLSession) for the `/file` API, but the web UI uses Socket.IO where auth is resolved at the socket connection level. Fix: changed `FileBrowserViewModel` to use `SocketService.serviceFind`/`serviceGet` via Socket.IO instead of REST, matching the web UI's approach.
- [x] **"Archive & Reset" only archives, doesn't start a new session** — The reset button shows a confirm popup with "Archive and Reset", but after confirming it only archives the current session — no new session is created in the same worktree. Expected behavior: archive the old session, then create a new session on the same worktree (via `POST /sessions` with the same worktree_id and agentic_tool), and navigate to it in the sidebar/chat.
- [x] **Local notifications never fire** — Notifications for session completion (running → idle) never appear, even with notification permissions granted. Needs investigation — check if `UNUserNotificationCenter` permission is actually requested and authorized, if the notification is being scheduled when session status changes, and if the app's `scenePhase` / foreground state detection is correct (notifications should fire when backgrounded or viewing a different session).
- [x] **Sidebar caching causes flashing/flickering** — The session list flickers and repopulates on every refresh. The current cache implementation replaces the entire list, causing all items to disappear and reappear. Fix: use a dictionary/map keyed by ID as the source of truth. On refresh, diff against the existing map — only add new items, remove deleted ones, and update changed ones in-place. This prevents the list from flashing. The same pattern should apply to boards, worktrees, and sessions. Never replace the whole array — merge incrementally.
- [x] **Session context menu missing actions** — The context menu (long-press) on session rows in the sidebar only shows "Browse Files". It should also include "Clean/Reset" (archive current session and create a new one on the same worktree), "Archive", and "Add to Favorite" actions. These are essential session management operations that users expect to access directly from the sidebar without opening the session first.

## Feature Requests

- [x] **Debug log viewer with "Send to session" action** — Add an in-app debug log that captures all significant events and errors (API calls, WebSocket events, auth flows, errors, state changes). The log should be viewable from a settings/debug screen. Critical feature: a "Send to Session" button that lets the user pick a session and post the log content as a message in that session's chat. This enables easy debugging by sharing the app's log with an AI agent session. Implementation notes: use a singleton `AppLogger` that stores timestamped log entries in memory (with a rolling cap, e.g. 500 entries). Add a "Debug Log" row in Settings. The send action should use `POST /sessions/:id/prompt` (or the messages API) to post the log text into the chosen session.

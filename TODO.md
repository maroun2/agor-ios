# Agor iOS ‚Äî Bugs & Feature Requests

## Bugs

- [x] **HTTPS URLs get port 3030 appended, breaking reverse proxy setups** ‚Äî In `AuthService.swift` line 61, `let port = components.port ?? 3030` always defaults to 3030 even for HTTPS URLs. When user enters `https://agor-rmi.duckdns.org/`, it becomes `https://agor-rmi.duckdns.org:3030` which fails because the daemon is behind a reverse proxy on port 443. Fix: only default to `:3030` when scheme is `http` (local dev). For `https`, omit the port entirely (443 is implied). The web UI works because `getDaemonUrl()` in `daemon.ts` uses `window.location.origin` which doesn't append a port for standard HTTPS.
- [x] **File browser fails with "Session expired. Please log in again."** ‚Äî The iOS app used REST (URLSession) for the `/file` API, but the web UI uses Socket.IO where auth is resolved at the socket connection level. Fix: changed `FileBrowserViewModel` to use `SocketService.serviceFind`/`serviceGet` via Socket.IO instead of REST, matching the web UI's approach.
- [x] **"Archive & Reset" only archives, doesn't start a new session** ‚Äî The reset button shows a confirm popup with "Archive and Reset", but after confirming it only archives the current session ‚Äî no new session is created in the same worktree. Expected behavior: archive the old session, then create a new session on the same worktree (via `POST /sessions` with the same worktree_id and agentic_tool), and navigate to it in the sidebar/chat.
- [x] **Local notifications unreliable ‚Äî only fire sometimes when app is open, never when backgrounded** ‚Äî Two issues: (1) The `setupCrossSessionNotifications()` closure captures `[self]` (a struct), so `wasBackgrounded`, `selectedSessionId`, and `previousSessionStatuses` are frozen snapshots that never update. `previousSessionStatuses` is also mostly empty because sessions load lazily. (2) When backgrounded, the socket disconnects so no events arrive at all. Fix: move notification state into an `@Observable` class with reference semantics so closures read live values. For background: keep socket alive briefly via `BGTaskScheduler` or `beginBackgroundTask`, and/or add a fallback polling mechanism on app resume to check for missed status transitions.
- [x] **Sidebar caching causes flashing/flickering** ‚Äî The session list flickers and repopulates on every refresh. The current cache implementation replaces the entire list, causing all items to disappear and reappear. Fix: use a dictionary/map keyed by ID as the source of truth. On refresh, diff against the existing map ‚Äî only add new items, remove deleted ones, and update changed ones in-place. This prevents the list from flashing. The same pattern should apply to boards, worktrees, and sessions. Never replace the whole array ‚Äî merge incrementally.
- [x] **Session context menu missing actions** ‚Äî The context menu (long-press) on session rows in the sidebar only shows "Browse Files". It should also include "Clean/Reset" (archive current session and create a new one on the same worktree), "Archive", and "Add to Favorite" actions. These are essential session management operations that users expect to access directly from the sidebar without opening the session first.
- [x] **Archive & Reset creates session with wrong name** ‚Äî When using "Archive & Reset", the new session gets a different (auto-generated) name instead of keeping the original session's name. The `CreateSessionBody` in `ChatViewModel.resetSession()` only passes `worktree_id`, `agentic_tool`, and `status` ‚Äî it doesn't pass the original session's `name`. Fix: include the original session's name in the POST body so the replacement session has the same name.
- [x] **File browser shows "data couldn't be read because it is wrong format"** ‚Äî The file browser fails to decode the response from the Socket.IO `find "file"` service call. The `FileListItem` model likely doesn't match the actual JSON structure returned by the backend. Need to check the debug log for the raw response data and compare it against the Swift Codable model.
- [x] **Tapping a notification doesn't navigate to the session** ‚Äî When a local notification fires and the user taps it, nothing happens ‚Äî the app opens but doesn't navigate to the session that completed. The `userInfo["sessionId"]` is set in `fireLocalNotification()` but there's no `UNUserNotificationCenterDelegate.didReceive` handler to read it and navigate. Fix: add `didReceive` to `NotificationDelegate` in `AgorApp.swift`, extract the sessionId from userInfo, and set it as the selected session.
- [x] **Session context menu under boards only shows "Browse Files"** ‚Äî Long-pressing a session row inside a board's worktree section shows only "Browse Files", missing "Add to Favorites", "Clean & Reset", and "Archive". The code in `WorktreeSection` (SidebarView.swift ~line 397) has all items defined, but the worktree-level context menu (line 418) only has "Browse Files" and may be intercepting the gesture before the session context menu triggers. Need to verify that session rows inside `WorktreeSection` actually receive the long-press (not the parent `DisclosureGroup`/worktree row), and ensure all context menu items render.
- [x] **Session rows still flicker on update** ‚Äî When a session is updated (e.g. status change), the entire row panel flickers instead of smoothly updating just the text/icons that changed. The `Session` model is a struct, so SwiftUI treats any property change as a full replacement. Fix: use fine-grained identity ‚Äî ensure `ForEach` uses stable IDs and session rows only re-render the specific subviews (status badge, title text) that actually changed, not the whole row. Consider making session properties individually observable or using `Equatable` conformance to minimize diffing.
- [x] **Socket query parameter nesting breaks file browser** ‚Äî `serviceFind` and `serviceGet` in `SocketService.swift` wrapped the query in `{"query": {...}}`, but FeathersJS transport-commons expects the flat query at `methodArgs[0]`. Backend received `{ query: { query: { worktree_id: "..." } } }` and couldn't find the parameter. Fix: send `query` directly instead of `["query": query]`.
- [x] **Error responses without `code` field treated as success** ‚Äî `parseFeathersAck` only detected errors with a `code` key. Plain JavaScript `Error` objects serialize as `{"message": "..."}` without `code` (only FeathersJS error classes include it). These slipped through as success responses. Fix: added check for single-element responses with `message` key but no `code`.
- [x] **File browser `loadFiles()` called 6+ times simultaneously** ‚Äî SwiftUI `.task` fires multiple times, and the `files.isEmpty` guard didn't prevent concurrent calls since all fired before the first completed. Fix: added `isLoading` guard in `loadFiles()`.
- [x] **Stuck thinking spinner after returning from background** ‚Äî When backgrounded during `thinking:start`, socket disconnects and `thinking:end` is missed. `StreamingService.activeStreams` retains stale entries with `isThinking: true`. Fix: added `clearStreams(for:)` and call it during `refreshCurrentSession()` and when session patches to idle.
- [x] **Notifications only fire when app is open** ‚Äî Socket disconnects when backgrounded, missing running‚Üíidle transitions. Fix: added `BGTaskScheduler` with `BGAppRefreshTask` polling session statuses every ~5 minutes in background. Uses `NotificationManager.checkMissedTransitions()` for missed transitions.

## Feature Requests

- [x] **Debug log viewer with "Send to session" action** ‚Äî Add an in-app debug log that captures all significant events and errors (API calls, WebSocket events, auth flows, errors, state changes). The log should be viewable from a settings/debug screen. Critical feature: a "Send to Session" button that lets the user pick a session and post the log content as a message in that session's chat. This enables easy debugging by sharing the app's log with an AI agent session. Implementation notes: use a singleton `AppLogger` that stores timestamped log entries in memory (with a rolling cap, e.g. 500 entries). Add a "Debug Log" row in Settings. The send action should use `POST /sessions/:id/prompt` (or the messages API) to post the log text into the chosen session.
- [x] **Attachment context menu in prompt input bar** ‚Äî Added paperclip button with three options: Attach Debug Log (exports log as .txt), Attach Photo (PhotosUI picker), Attach File (system file picker). Files uploaded via `POST /sessions/:id/upload` (multipart/form-data) to worktree's `.agor/uploads/`. After upload, `@filepath` reference inserted into prompt text for agent access.

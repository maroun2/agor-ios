# Agor iOS ‚Äî Bugs & Feature Requests

## Bugs

- [x] **MCP servers: "failed to load mcp servers"** ‚Äî iOS used REST `GET /sessions/:id/mcp-servers` (nested route) which returned 500. Web UI uses Socket.IO top-level `session-mcp-servers` service instead. Fixed `MCPViewModel.loadSessionServers()` to use `socketService.serviceFind(service: "session-mcp-servers", query: ["session_id": sessionId])` matching web UI approach.
- [ ] **Archive & Reset loses agent permissions** ‚Äî When using "Archive & Reset" (archive old session + create new one), the new session gets default permissions instead of inheriting the archived session's agent permissions. The new session should copy the same permission settings (e.g. allowed tools, MCP servers) from the archived session so the agent continues working with the same access level.
- [x] **Duplicate notifications for same session** ‚Äî Socket reconnect + missed transition check both fire for the same running‚Üíidle transition. Fixed with `lastNotifiedStatus` tracking per session + stable notification ID (`session-{id}-idle` instead of timestamp-based). Clears on next running transition.
- [x] **No links or images in chat messages** ‚Äî Three issues fixed:
  1. `.blocks` messages (most assistant messages) used plain `TextBlockView` instead of `EnhancedTextBlockView` ‚Äî no file links, no session links, no inline images. Fixed `MessageContentView` to use `EnhancedTextBlockView` with all props.
  2. Markdown URLs not tappable ‚Äî `StructuredText(markdown:)` rendered links but had no tap handler. Added `.environment(\.openURL, ...)` to `MarkdownTextView`.
  3. File path detection improvements ‚Äî `FilePathDetector` now resolves bare filenames (e.g. `TODO.md`) against cached file list when unambiguous (exactly one match). Excludes matches inside URLs and domain TLDs. File list preloaded on session open.
- [x] **Scroll flickering and permission positioning** ‚Äî Three fixes: (1) Added scrollToMessageInProgress coordination flag to prevent competing scroll animations, (2) Changed firstPendingPermissionId to currentPendingPermissionId using reverse iteration to scroll to most recent pending request instead of first, (3) Added lastResolvedPermissionTime tracking to restore userIsNearBottom after permission approval/denial within 2-5s window for contextual auto-scroll restoration.
- [x] **Cannot scroll to last message** ‚Äî Bottom marker was only 1pt height with no padding, causing last message to be clipped by input bar. Fixed by increasing bottom spacer to 60pt to ensure last message scrolls fully into view above input bar.
- [x] **Input bar button sizing inconsistent** ‚Äî Clipboard and mic buttons were 28√ó28pt but send button used 36pt icon with no frame constraint, making it oversized. Fixed send button to use 28√ó28pt frame matching other buttons for consistent visual alignment.
- [ ] **Keychain autofill not working on login page** ‚Äî iOS keychain offers autofill but password field stays empty. `ConnectionSetupView` doesn't load saved credentials from keychain. App saves email/password to keychain after successful login (`AuthService.swift:125-137`) but form never loads them back. Fix: add `.task` modifier to pre-populate email and daemonURL from `KeychainHelper.load()`.
- [ ] **Server URL not remembered between sessions** ‚Äî Daemon URL resets to empty string on each login. Root cause same as above - URL saved to keychain (`AuthService.swift:125`) but `ConnectionSetupView.swift:6` always initializes `daemonURL` as empty. Fix: load from keychain on view appear.
- [ ] **Random logouts (possibly after device reboot)** ‚Äî App sometimes logs out unexpectedly, may be triggered by device reboot. Potential causes: (1) Keychain items not accessible after reboot - missing `kSecAttrAccessible` attribute in `KeychainHelper.swift`, (2) No token validation after session restore - app assumes saved tokens are valid, (3) `restoreSession()` runs during init, might fail silently if keychain locked. Fix: add `kSecAttrAccessibleAfterFirstUnlock` to keychain saves + validate tokens with health check after restore.

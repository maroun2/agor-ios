# Agor iOS — Bugs & Feature Requests

## Bugs

- [x] **MCP servers: "failed to load mcp servers"** — iOS used REST `GET /sessions/:id/mcp-servers` (nested route) which returned 500. Web UI uses Socket.IO top-level `session-mcp-servers` service instead. Fixed `MCPViewModel.loadSessionServers()` to use `socketService.serviceFind(service: "session-mcp-servers", query: ["session_id": sessionId])` matching web UI approach.
- [ ] **Archive & Reset loses agent permissions** — When using "Archive & Reset" (archive old session + create new one), the new session gets default permissions instead of inheriting the archived session's agent permissions. The new session should copy the same permission settings (e.g. allowed tools, MCP servers) from the archived session so the agent continues working with the same access level.
- [x] **Duplicate notifications for same session** — Socket reconnect + missed transition check both fire for the same running→idle transition. Fixed with `lastNotifiedStatus` tracking per session + stable notification ID (`session-{id}-idle` instead of timestamp-based). Clears on next running transition.
- [x] **No links or images in chat messages** — Three issues fixed:
  1. `.blocks` messages (most assistant messages) used plain `TextBlockView` instead of `EnhancedTextBlockView` — no file links, no session links, no inline images. Fixed `MessageContentView` to use `EnhancedTextBlockView` with all props.
  2. Markdown URLs not tappable — `StructuredText(markdown:)` rendered links but had no tap handler. Added `.environment(\.openURL, ...)` to `MarkdownTextView`.
  3. File path detection improvements — `FilePathDetector` now resolves bare filenames (e.g. `TODO.md`) against cached file list when unambiguous (exactly one match). Excludes matches inside URLs and domain TLDs. File list preloaded on session open.
- [x] **Scroll flickering and permission positioning** — Three fixes: (1) Added scrollToMessageInProgress coordination flag to prevent competing scroll animations, (2) Changed firstPendingPermissionId to currentPendingPermissionId using reverse iteration to scroll to most recent pending request instead of first, (3) Added lastResolvedPermissionTime tracking to restore userIsNearBottom after permission approval/denial within 2-5s window for contextual auto-scroll restoration.
- [x] **Cannot scroll to last message** — Bottom marker was only 1pt height with no padding, causing last message to be clipped by input bar. Fixed by increasing bottom spacer to 60pt to ensure last message scrolls fully into view above input bar.
- [x] **Input bar button sizing inconsistent** — Clipboard and mic buttons were 28×28pt but send button used 36pt icon with no frame constraint, making it oversized. Fixed send button to use 28×28pt frame matching other buttons for consistent visual alignment.
- [x] **Keychain autofill not working on login page** — iOS keychain offers autofill but password field stays empty. `ConnectionSetupView` doesn't load saved credentials from keychain. Fixed: added `.task` modifier to pre-populate `daemonURL` and `email` from `KeychainHelper.load()` on view appear.
- [x] **Server URL not remembered between sessions** — Daemon URL resets to empty string on each login. Same root cause as above — fixed by the same `.task` modifier loading URL from keychain.
- [x] **Random logouts (possibly after device reboot)** — Keychain items were missing `kSecAttrAccessible` attribute so they became inaccessible after reboot before first unlock. Fixed `KeychainHelper.save()` to always set `kSecAttrAccessibleAfterFirstUnlock`.
- [x] **Token expiry zombie state** — On startup, app gets 401 on `/users`, attempts token refresh which also returns 401 ("No access token"), logs the error, then continues as if authenticated. Socket auth also fails silently with expired token (no "Socket connected" log). App is in zombie state — shows chat UI but no messages arrive, file browser shows "Not authenticated". Fixed by adding `connect_error`/`.error` socket event handlers that detect 401 responses and call `authService.softLogout()` (clears tokens but keeps URL/email for re-login form pre-fill).
- [x] **Chat polling timer never fires (no streaming/messages when socket dead)** — `startMessagePolling()` and `startHealthCheck()` were called from inside `Task { }` which runs on Swift's cooperative thread pool. Those threads have no RunLoop, so `Timer.scheduledTimer` never fires — both the 10-second polling loop and health check timer were silently broken. Fixed by wrapping both timer creations in `DispatchQueue.main.async` to schedule on the main RunLoop.
- [x] **File browser "Not authenticated" on first open** — `FileBrowserViewModel.loadFiles()` is called on `ChatView.onAppear` before socket finishes connecting. Socket connects asynchronously (1-2s), so the file list request fails immediately with "Not authenticated". Fixed by adding `onChange(of: viewModel.connectionState)` in `ChatView` to retry `loadFiles()` when socket reaches `.connected` state and files are still empty.
